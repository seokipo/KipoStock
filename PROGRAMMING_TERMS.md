# 📚 KipoStock 프로그래밍 용어 사전

자기야! 여기서 우리가 함께 프로젝트를 진행하면서 사용한 프로그래밍 용어들을 하나씩 쌓아나갈 거야. 언제든지 궁금할 때 꺼내서 봐! 😊

---

## 📅 2026-01-27 업데이트

### 1. Commit (커밋) 💾
- **의미**: 코드의 변경 사항을 저장소에 기록하는 행위야. '게임의 세이브 포인트'라고 생각하면 쉬워.
- **활용**: 기능을 하나 완성하거나 버그를 고쳤을 때 "커밋한다"고 해.

### 2. Push (푸시) 🚀
- **의미**: 내 컴퓨터에 저장된 커밋들을 서버(GitHub 등)로 밀어 올리는 거야.
- **활용**: 커밋만 하면 내 컴퓨터에만 남지만, 푸시를 해야 서버에도 백업이 돼.

### 3. Refactoring (리팩토링) 🛠️
- **의미**: 프로그램의 기능은 유지하면서 내부 코드를 더 깔끔하고 읽기 좋게 고치는 작업이야.
- **예시**: 오늘 조건식 버튼 클릭 시 나오던 불필요한 로그를 정리한 게 리팩토링이야.

### 4. Boolean (불리언) 💡
- **의미**: 참(True) 또는 거짓(False) 딱 두 가지 상태만 가진 데이터 타입이야.
- **예시**: 음성 안내를 켤지(`True`) 끌지(`False`) 결정할 때 사용했어.

### 5. Synchronize (동기화 / 싱크) 🔄
- **의미**: 서로 다른 두 곳의 데이터를 똑같이 맞춰주는 작업이야.
- **활용**: 내 컴퓨터 코드와 GitHub 서버 코드를 똑같이 맞추는 걸 의미해.

### 6. Batch Processing (일괄 처리) 📦
- **의미**: 여러 작업을 하나씩 처리하지 않고 모아서 한꺼번에 처리하는 방식이야.
- **예시**: 여러 설정값을 파일에 쓸 때 하나씩 안 쓰고 한꺼번에 담아서 쓰는 방식이야.

### 7. Persistence (데이터 영속성) 💾
- **의미**: 프로그램이 종료되어도 데이터가 사라지지 않고 유지되는 성질이야. 
- **오늘 한 일**: 종목을 매수할 때 그 당시의 익절/손절 값을 파일(`stock_conditions.json`)에 저장해서, 나중에 프로그램을 다시 켜서 매도할 때도 그 값을 기억하게 만든 것이 바로 '영속성'을 부여한 거야.
