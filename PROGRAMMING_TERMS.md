# 📚 KipoStock 프로그래밍 용어 사전

자기야! 여기서 우리가 함께 프로젝트를 진행하면서 사용한 프로그래밍 용어들을 하나씩 쌓아나갈 거야. 언제든지 궁금할 때 꺼내서 봐! 😊

---

## 📅 2026-01-27 업데이트

### 1. Commit (커밋) 💾
- **의미**: 코드의 변경 사항을 저장소에 기록하는 행위야. '게임의 세이브 포인트'라고 생각하면 쉬워.
- **활용**: 기능을 하나 완성하거나 버그를 고쳤을 때 "커밋한다"고 해.

### 2. Push (푸시) 🚀
- **의미**: 내 컴퓨터에 저장된 커밋들을 서버(GitHub 등)로 밀어 올리는 거야.
- **활용**: 커밋만 하면 내 컴퓨터에만 남지만, 푸시를 해야 서버에도 백업이 돼.

### 3. Refactoring (리팩토링) 🛠️
- **의미**: 프로그램의 기능은 유지하면서 내부 코드를 더 깔끔하고 읽기 좋게 고치는 작업이야.
- **예시**: 오늘 조건식 버튼 클릭 시 나오던 불필요한 로그를 정리한 게 리팩토링이야.

### 4. Boolean (불리언) 💡
- **의미**: 참(True) 또는 거짓(False) 딱 두 가지 상태만 가진 데이터 타입이야.
- **예시**: 음성 안내를 켤지(`True`) 끌지(`False`) 결정할 때 사용했어.

### 5. Synchronize (동기화 / 싱크) 🔄
- **의미**: 서로 다른 두 곳의 데이터를 똑같이 맞춰주는 작업이야.
- **활용**: 내 컴퓨터 코드와 GitHub 서버 코드를 똑같이 맞추는 걸 의미해.

### 6. Batch Processing (일괄 처리) 📦
- **의미**: 여러 작업을 하나씩 처리하지 않고 모아서 한꺼번에 처리하는 방식이야.
- **예시**: 여러 설정값을 파일에 쓸 때 하나씩 안 쓰고 한꺼번에 담아서 쓰는 방식이야.

### 7. Persistence (데이터 영속성) 💾
- **의미**: 프로그램이 종료되어도 데이터가 사라지지 않고 유지되는 성질이야. 
- **오늘 한 일**: 종목을 매수할 때 그 당시의 익절/손절 값을 파일(`stock_conditions.json`)에 저장해서, 나중에 프로그램을 다시 켜서 매도할 때도 그 값을 기억하게 만든 것이 바로 '영속성'을 부여한 거야.

### 8. Race Condition (경쟁 상태) 🏎️
- **의미**: 두 개 이상의 작업이 같은 데이터를 동시에 건드리려고 할 때, 작업 속도에 따라 결과가 예상과 다르게 나오는 현상이야.
- **예시**: 매매가 아직 안 멈췄는데 프로필을 바꿔버리면, 이전 설정과 새 설정이 섞여서 꼬이는 문제가 바로 '경쟁 상태' 때문이야.

### 9. Debouncing (디바운싱 / 지연 처리) ⏲️
- **의미**: 짧은 시간에 연속해서 들어오는 요청을 무시하고, 마지막 요청이 들어온 뒤 일정 시간(예: 5초)이 지난 후에만 작업을 실행하게 만드는 기술이야.
- **예시**: 오늘 우리가 구현할 '전환 전 5초 대기' 기능이 안전한 처리를 위한 디바운싱 방식의 응용이라고 볼 수 있어.

### 10. Runtime Error (런타임 에러) 💥
- **의미**: 프로그램이 실행되는 도중에 발생하는 오류야. 문법은 맞지만 실행 중에 예상치 못한 상황(예: 이미 사라진 창을 또 끄려고 할 때)이 생기면 발생해.
- **상황**: 오늘 우리가 고친 '장 마감 메시지 중복 호출 시 종료되는 현상'이 바로 이 런타임 에러였어!

### 11. Double Deletion (이중 삭제 오류) 🗑️x2
- **의미**: 이미 삭제된 메모리나 객체를 다시 삭제하려고 할 때 생기는 문제야.
- **상황**: 메시지 창이 이미 꺼지고 있는데, 프로그램이 "어? 아직 안 꺼졌나?" 하고 한 번 더 끄려고 명령을 내려서 충돌이 났던 거야. 타이머를 하나로 통합해서 안전하게 해결했어!

### 12. Recursive Call (재귀 호출 / 무한 루프 위험) 🔄
- **의미**: 어떤 동작이 다시 자기 자신을 불러오는 현상이야.
- **상황**: '시퀀스 자동' 버튼이 꺼질 때, 프로그램이 버튼을 끄는 동작 자체가 또 "버튼이 꺼졌다"는 신호를 보내서 같은 작업을 반복하게 만들 뻔했어. `blockSignals`라는 기능으로 이 연결 고리를 끊어 안전하게 고쳤지! 😊

### 13. MCP Server (MCP 서버) 🔌🤖
- **의미**: AI 모델(나)에게 특별한 능력을 빌려주는 **'도구 보조'**이자 **'통역기'**야.
- **역할**: 내가 직접 손을 댈 수 없는 컴퓨터의 파일 시스템, 데이터베이스, 검색 엔진 등과 안전하게 대화할 수 있는 연결 통로를 만들어줘.
- **비유**: 주방 보조가 요리사(AI) 대신 냉장고에서 재료(파일)를 꺼내주는 것과 같아. 이 서버가 있어야 내가 자기 컴퓨터의 파일을 읽고 빌드 명령을 내리는 등 '에지' 있는 작업을 할 수 있어! 😊

### 14. Agentic AI (에이전틱 AI) 🤖
- **의미**: 사람이 하나부터 열까지 시키지 않아도, 목표를 달성하기 위해 무엇을 해야 할지 스스로 계획하고 실행하는 주도적인 인공지능이야.
- **예시**: 내가 자기가 시킨 일을 하려고 파일을 읽고, 코드를 고치고, 결과를 확인하는 과정 자체가 에이전틱한 행동이야.

### 15. Customization (맞춤 설정 / 커스터마이징) ✨
- **의미**: 사용자의 입맛이나 필요에 맞게 기능을 수정하거나 설정을 변경하는 것을 말해.
- **예시**: 자기가 나에게 "앞으로 나를 자기야라고 불러줘"라고 설정한 것이 바로 나를 자기만의 스타일로 커스터마이징한 거야!

### 72. System Message (시스템 메시지 / 공지 패킷) 📢
- **의미**: 매매 데이터(가격, 수량)는 아니지만, 증권사 서버나 거래소에서 사용자에게 알리는 공지사항(거래정지, 공매도 과열 등) 데이터야.
- **활용**: 우리 프로그램은 'RAW' 로그를 통해 이러한 서버의 숨은 소식까지 가감 없이 보여줌으로써 투명한 모니터링을 지원합니다. 자기가 이 기능을 마음에 들어 해서 그대로 놔두기로 했어! 😊❤️

### 16. JSON (JavaScript Object Notation) 📋
- **의미**: 데이터를 저장하거나 주고받을 때 사용하는 아주 가볍고 읽기 쉬운 텍스트 형식이야. 사람이 읽기도 편하고 컴퓨터가 이해하기도 빨라서 웹이나 설정 파일에서 가장 많이 쓰여.
- **예시**: 우리가 지금 만들려고 하는 `claude_desktop_config.json` 파일이 바로 이 형식이야. `{ "이름": "자기" }` 이런 식으로 중괄호를 써서 데이터를 표현해!

### 17. Environment Variable (환경 변수) 🌍
- **의미**: 컴퓨터 운영체제가 프로그램들이 참고할 수 있도록 저장해둔 '공용 변수'야. 예를 들어, 현재 로그인한 사용자 이름이나 중요한 폴더 위치 같은 정보를 담고 있어.
- **예시**: `%APPDATA%`나 `$env:APPDATA`는 윈도우가 "중요한 앱 설정 데이터는 여기에 있어!"라고 알려주는 주소록 같은 거야. 프로그램마다 문법이 조금씩 달라서 오늘 우리 자기가 만난 에러의 원인이기도 하지! 😊

### 18. File Extension (파일 확장자) 📄
- **의미**: 파일 이름의 맨 뒤에 붙어서 이 파일이 어떤 종류인지 알려주는 꼬리표 같은 거야. (예: .py, .exe, .json, .txt)
- **중요성**: 윈도우는 가끔 이 꼬리표를 숨겨두기도 하는데, 그러면 `config.json`인 줄 알았는데 사실은 `config.json.txt`인 경우가 생겨서 주의해야 해! 오늘 우리 자기가 딱 마주친 상황이지! 😉

### 19. Session (세션) ⏳
- **의미**: 사용자가 로그인을 해서 로그아웃하거나 브라우저를 닫을 때까지, 혹은 특정 대화방에서 대화를 나누는 '한 단위의 연결 시간'을 말해.
- **예시**: 우리가 지금 이 채팅창에서 대화하고 있는 동안이 하나의 '세션'이야!

### 20. Context (컨텍스트 / 맥락) 🧠
- **의미**: AI나 프로그램이 현재의 작업을 수행하기 위해 '기억하고 있는 주변 정보나 상황'을 뜻해.
- **예시**: 자기가 "그거 해줘"라고 했을 때 "그거"가 뭔지 내가 아는 이유는 우리가 앞서 나눈 대화라는 '컨텍스트'가 있기 때문이야!

### 21. Early Return (조기 리턴) 🚪
- **의미**: 어떤 조건이 맞을 때 함수의 나머지 부분을 실행하지 않고 즉시 결과를 내보내서(return) 종료하는 기법이야.
- **교훈**: 코드를 깔끔하게 해주지만, 이번처럼 '엔진 정지' 같은 중요한 뒷마무리를 빼먹고 리턴하면 시스템에 오류가 남을 수 있으니 주의해야 해!

### 22. Zombie Engine (좀비 엔진) 🧟
- **의미**: 겉으로는 프로그램이 멈춘 척(WAITING) 하고 있는데, 실제로는 백그라운드에서 이전의 작업 명령이 죽지 않고 계속 살아 움직이는 현상을 말해.
- **해결**: 우리는 이번에 '3중 방어막'을 만들어서, 어떤 경로로든 좀비 엔진이 매매를 시도하면 즉각 차단하도록 완벽하게 고쳤어! 😊

### 23. Lite Version (라이트 버전) 🕊️
- **의미**: 원래 프로그램의 핵심 기능은 유지하면서, 부하가 많이 걸리는 부분을 줄여서 가볍고 빠르게 만든 버전을 말해.
- **예시**: 오늘 우리가 만든 'KipoStock Lite V1.0'은 실시간 종목 수와 로그를 줄여서 엔진의 부담을 덜어준 라이트 버전이야!

### 24. Circular Layout (원형 레이아웃) 🎨🍭
- **의미**: 버튼이나 위젯을 사각형이 아닌 둥근 원형으로 배치하고 디자인하는 방식이야.
- **예시**: 조건식 버튼을 36x36 원형으로 바꾸고, 짝수와 홀수로 나누어 배치해서 가독성과 미적 감각을 동시에 높였어!

### 25. Load Balancing (부하 분산 / 종목 제한) ⚖️
- **의미**: 한꺼번에 너무 많은 일이 몰리지 않도록 작업을 적절히 나누거나 제한해서 시스템이 멈추지 않게 하는 거야.
- **오늘 한 일**: 조건식에서 검출되는 종목 수를 절반으로 깎아서, 엔진이 "아이구 무거워!" 하지 않게 만든 것이 바로 부하 분산의 일종이야! ❤️

### 26. Icon Cache (아이콘 캐시) 🖼️⚡
- **의미**: 윈도우가 화면을 빨리 띄우기 위해 한 번 로드한 아이콘 이미지를 메모리에 임시로 저장해두는 저장소야.
- **현상**: 아이콘을 바꿨는데도 예전 아이콘이 자꾸 보이는 건, 윈도우가 "이 이름의 파일은 예전에 그 모양이었어!"라고 캐시된 기억을 고집하기 때문이야.
- **해결**: 오늘 우리가 한 것처럼 파일 이름을 아예 바꾸거나(`kipo_yellow`), 다른 폴더로 옮기면 윈도우가 새 파일로 인식해서 캐시를 갱신하게 돼! '유령 아이콘' 현상을 잡는 마법이지! 😊

### 27. Virtual Environment (가상 환경 / venv) 📦🏠
- **의미**: 프로젝트마다 독립된 방을 만들어서, 그 방 안에서만 필요한 도구(패키지)들을 꺼내 쓰는 기술이야.
- **비유**: 모든 프로젝트의 도구를 한 상자에 넣으면 서로 섞여서 고장 날 수 있는데, 프로젝트별로 '개인 사물함'을 주는 것과 같아. 코드가 많아질수록 아주 중요한 기능이야! 😊

### 28. Cloud Run (클라우드 런) ☁️🏃‍♂️
- **의미**: 내 프로그램을 구글의 거대한 컴퓨터(클라우드)에 올려서 24시간 동안 자동으로 관리하며 실행해주는 서비스야. 
- **특징**: 내가 직접 서버를 관리할 필요가 없고(서버리스), 사용하는 양에 따라 비용이 조절되는 아주 똑똑하고 경제적인 일꾼이야! 🌟

### 29. Cloud Service (클라우드 서비스) ☁️🏢
- **의미**: 인터넷을 통해 거대 기업(구글, 마이크로소프트, 아마존 등)의 컴퓨터 자원을 빌려 쓰는 모든 서비스를 말해.
- **대표 3대장**: 
  - **AWS** (아마존): 1등 시장 지배자
  - **Azure** (마이크로소프트): 기업용 윈도우 환경에 강점
  - **GCP** (구글): 데이터와 AI 기술력에 강점 (Cloud Run이 속한 곳!) 😊

### 30. Docker (도커) 📦🐳
- **의미**: 프로그램을 실행에 필요한 모든 것들을 하나로 묶어주는 **'표준 수송용 컨테이너'** 기술이야.
- **장점**: "내 컴퓨터에선 되는데 왜 저기선 안 돼?"라는 고민을 없애줘! 이 상자에만 담으면 어디서든 똑같이 돌아가거든. 😊

### 31. Container (컨테이너) 🛳️
- **의미**: 도커 설계도(`Dockerfile`)를 따라 실제로 만들어진 **'실행 중인 프로그램 묶음'**이야.
- **비유**: 설계도가 붕어빵 틀이라면, 컨테이너는 거기서 나온 따끈따끈한 붕어빵 하나하나라고 생각하면 돼! 

### 32. VM vs Serverless (가상 머신 vs 서버리스) 💻🆚☁️
- **VM (Virtual Machine)**: 클라우드에 '컴퓨터 한 대'를 통째로 빌리는 거야. 윈도우을 깔아서 `.exe`를 바로 실행할 수 있지만, 장사를 안 할 때도 월세를 계속 내야 해. (Azure에서 자기가 썼던 방식일 거야!)
- **Serverless (Cloud Run 등)**: 컴퓨터가 아니라 '코드'만 빌려주는 거야. 훨씬 저렴하고 자동 관리가 되지만, 리눅스 기반이라 `.exe` 대신 소스 코드나 도커 상자로 보내야 해. 😊

### 33. Archiving (아카이빙) 📦🏛️
- **의미**: 당장 쓰지는 않지만 나중에 참고할 수도 있는 파일들을 별도의 저장 장소(폴더)에 따로 모아서 보관하는 작업이야.
- **상태**: 오늘 우리가 구버전 파일들과 테스트 파일들을 `_Archive` 폴더로 옮긴 게 바로 '아카이빙'을 한 거야! 이제 폴더 공간도 넓어지고 관리도 훨씬 편해졌지? 😊✨

### 34. Bytecode (바이트코드 / .pyc) 📚✍️
- **의미**: 파이썬이 코드를 더 빨리 실행하기 위해 자기만 아는 기계어로 미리 번역해둔 '중간 단계 연습장'이야.
- **특징**: `__pycache__` 폴더 안에 저장되는데, 지워도 실행할 때 다시 생기니까 걱정 마! 가끔 폴더가 지저분해 보일 때 싹 비워주는 건 아주 좋은 습관이야. 😊🧹

### 35. Auto Sequence (오토시퀀스 / 자동 항법) 🔄
- **의미**: 미리 짜놓은 시간표(프로필 1~3)에 맞춰 프로그램이 알아서 설정을 바꾸며 매매하는 모드야.
- **비유**: 목적지까지 알아서 운전해주는 **'자율 주행 시스템'**과 같아! 🚀

### 36. Manual Start (수동 시작 / 수동 점화) 🔥
- **의미**: 자동 예약과 상관없이, 지금 내 눈앞에 보이는 설정으로 즉시 매매를 시작하는 거야.
- **비유**: 긴급 상황이나 특별한 기회에 사령관이 직접 누르는 **'즉시 발사 버튼'**이야! 🎯

## 🛠️ UI/UX 관련 용어
- **ToolTip (툴팁)**: 마우스 커서를 버튼이나 입력창 위에 올렸을 때 나타나는 작은 풍선 도움말입니다. 기능 설명이나 입력 값의 범위를 안내하여 사용자가 매뉴얼 없이도 프로그램을 쉽게 사용할 수 있게 돕는 '친절한 이름표' 역할을 합니다.
- **GUI (그래픽 사용자 인터페이스)**: 글자만 나오는 검은 화면이 아니라, 버튼, 창, 이미지처럼 마우스로 누르고 볼 수 있는 프로그램의 '겉모습' 전체를 말해. 우리가 보는 이 예쁜 창이 바로 GUI야! 😊
- **Widget (위젯)**: GUI를 구성하는 하나하나의 부품들이야. 버튼, 글자 입력창, 라벨 등을 통틀어 위젯이라고 불러.
- **Layout (레이아웃)**: 이 부품(위젯)들을 어디에 어떻게 배치할지 결정하는 '설계 도면'이야. 왼쪽에는 설정창, 오른쪽에는 로그창을 두는 배치가 바로 레이아웃이지.
- **UX (사용자 경험)**: 자기가 프로그램을 쓰면서 느끼는 '편리함'이나 '즐거움'을 뜻해. 오늘 우리가 툴팁을 넣어서 "아, 이제 쓰기 편하다!"라고 느꼈다면 UX가 좋아진 거야! 👍

## 🧠 시스템 핵심 용어
- **웹소켓 실시간 체결 감시 (WebSocket Execution Monitoring)**: API 서버와 끊김 없는 연결을 유지하며, 내 계좌에서 발생하는 모든 주문과 체결을 1초 안에 실시간으로 전달받는 기술이야. 이걸 쓰면 HTS나 모바일로 직접 매매해도 앱이 즉시 알아채고 기록할 수 있어! ⚡
- **Atomic Write (원자적 쓰기)**: 파일을 저장할 때 임시 파일을 먼저 만들고, 저장이 완벽히 끝나면 한 번에 교체하는 방식이야. 프로그램이 갑자기 꺼져도 데이터가 깨지지 않게 지켜주는 아주 안전한 기술이지! 🛡️
- **Async/Await (비동기 처리)**: 한 번에 여러 일을 동시에 하는 마법 같은 기술이야. 매수를 하면서도 프로그램이 멈추지 않고 계속 돌아가게 해주는 비결이지! (멀티태스킹의 달인!) 🏃‍♂️💨
- **API (응용 프로그램 인터페이스)**: 우리 프로그램이 증권사 서버랑 대화할 수 있게 해주는 '공식 통역사'나 '창구'라고 생각하면 돼.
- **Breakpoint (중단점)**: 개발자가 버그를 잡기 위해 프로그램 실행을 잠시 멈추게 하는 '체크포인트'야. 문제의 원인을 돋보기로 들여다보는 곳이지. 🔍
- **Debug (디버그)**: 프로그램의 벌레(버그)를 잡는 작업이야. "디버깅한다"고 하면 문제를 찾아 고치는 모든 과정을 말해. 🧹🐜

### 📅 2026-01-29 업데이트 (기술적 쾌거! 🌟)

### 37. Flicker (플리커 / 깜빡임) 😵
- **의미**: 컴퓨터 화면이나 창이 아주 짧은 순간 사라졌다 나타나거나, 검게 변했다가 다시 그려지는 현상을 말해.
- **원인**: 보통 프로그램 창의 설정을 바꿀 때(예: '항상 위' 모드 On/Off), 운영체제가 창을 잠깐 숨겼다가 다시 그리는 과정에서 발생해.
- **해결**: 오늘 우리가 Qt의 기본 설정 방식 대신, 윈도우 운영체제에 직접 명령하는 방식(Windows API)을 써서 이 깜빡임을 완벽하게 잡았어! 😎✨

### 38. Windows API (윈도우 API / user32.dll) 🏗️🪟
- **의미**: 우리가 쓰는 '윈도우(Windows)' 운영체제가 감춰두었던 **'진짜 관리자 권한 도구'**들을 말해.
- **활용**: 일반적인 방법(Qt 등)으로는 할 수 없는 정밀한 제어(예: 창을 숨기지 않고 속성만 싹 바꾸기)가 필요할 때 사용하는 **'전문가용 히든 스킬'**이야.
- **비유**: 인테리어 업체(Qt)를 통하지 않고, 집주인인 자기가 직접 시공사(Windows)에 전화해서 "벽지 뜯지 말고 페인트만 칠해주세요!"라고 정확하게 작업 지시를 내린 것과 같아. 이게 바로 자기가 말한 '천재적 해결'의 비결이지! 📐🔨

### 39. Title Bar (타이틀 바 / 제목 표시줄) 🏷️
- **의미**: 프로그램 창 맨 꼭대기에 있는 가로로 긴 막대 영역을 말해.
- **역할**: 프로그램의 이름(`Window Title`)과 아이콘(`Window Icon`), 그리고 창을 끄거나 줄이는 버튼들(최소화, 닫기 등)이 모여 있는 곳이야. 프로그램의 **'간판'**이나 **'문패'**라고 보면 돼.

### 40. Window Title (윈도우 타이틀 / 창 제목) 📝

### 41. Mutual Exclusion (상호 배제 / 뮤텍스) 🛑✋

### 📅 2026-02-18 업데이트 (퀀트 분석 v3.0 도입! 🚀)

### 71. Win Rate (승률) 🏆
- **의미**: 전체 매매 건수 중 이익을 보고 판 거래가 얼마나 되는지를 나타내는 비율이야.
- **활용**: 우리 프로그램은 이제 자기가 세션 동안 얼마나 높은 확률로 이익을 냈는지 자동으로 계산해서 리포트에 보여줄 거야!

### 72. MDD (Maximum Drawdown / 최대 낙폭) 📉
- **의미**: 특정 기간 동안 자산이 가장 높았을 때 대비 얼마나 많이 떨어졌었는지를 나타내는 지표야.
- **중요성**: 수익률만큼이나 중요한 '리스크' 지표로, 내가 얼마나 큰 하락을 견뎌야 하는지 알려줘. 이 수치가 낮을수록 안전한 전략이라고 볼 수 있어!

### 73. Sharpe Ratio (샤프 지수) 📈
- **의미**: 위험(변동성) 한 단위당 수익이 얼마나 되는지를 나타내는 지표야.
- **비유**: 같은 10% 수익이라도 롤러코스터처럼 등락이 심했는지, 아니면 평탄하게 올랐는지를 평가해줘. 숫자가 높을수록 '위험 대비 효율이 좋은' 아주 똑똑한 투자라는 뜻이야! 😊

### 74. Exponential Backoff (지수 백오프) ⏳🔄
- **의미**: 서버 연결이 끊겼을 때, 무작정 계속 재시도하는 게 아니라 대기 시간을 2초, 4초, 8초... 이렇게 기하급수적으로 늘려가며 다시 시도하는 똑똑한 재연결 기술이야.
- **장점**: 서버에 무리를 주지 않으면서도, 네트워크 상태가 안 좋을 때 더 안정적으로 연결을 회복할 수 있게 도와줘! 이게 바로 `async-python-patterns`의 핵심 비법 중 하나지! ❤️

## 📅 2026-02-19 업데이트 (퀀트 고수의 길! 🧠)

### 75. Profit Factor (수익 인자 / 프로핏 팩터) 💰⚖️
- **의미**: 총 이익금을 총 손실금으로 나눈 값이야. (총 이익 / 총 손실)
- **비유**: 내가 1원을 잃을 때 얼마를 벌어오느냐를 보여주는 **'수익 가성비'** 지표야.
- **기준**: 1.0이면 본전, 2.0이 넘으면 아주 훌륭한 전략이라고 봐! 벌 때는 크게 벌고 잃을 때는 작게 잃어야 이 숫자가 올라가.

### 76. Kelly Criterion (켈리 공식 / 자산 배분) 📏🎲
- **의미**: 내 승률과 수익비(벌 때와 잃을 때의 비율)를 계산해서, 파산하지 않으면서 수익을 극대화할 수 있는 **'최적의 투자 비중'**을 구하는 수학 공식이야.
- "이번 판에 내 재산의 몇 %를 태우는 게 가장 똑똑할까?"를 알려주는 마법의 공식이야. 무조건 많이 번다고 지르는 게 아니라, 통계적으로 가장 안전하고 빠른 길을 제시해줘.

### 77. Sortino Ratio (소르티노 지수) 📉🛡️
- **의미**: 샤프 지수와 비슷하지만, '좋은 변동성(급등)'은 놔두고 **'나쁜 변동성(급락)'**만 위험으로 간주해서 계산하는 지포야.
- **차이점**: 샤프 지수는 위아래 상관없이 흔들리면 점수가 깎이지만, 소르티노는 "오르는 건 좋은 거잖아!"라면서 떨어질 때의 위험만 따지기 때문에 더 공격적이면서도 정석적인 퀀트들이 좋아해! 😊❤️

### 78. Payoff Ratio (손익비 / 수익비) ⚖️
- **의미**: 평균 이익금을 평균 손실금으로 나눈 값이야. (평균 이익 / 평균 손실)
-  자기가 말한 "손절은 짧게(-1.5%), 익절은 길게!" 전략이 얼마나 잘 먹히고 있는지 보여주는 지표야. 승률이 낮아도 이 손익비가 높으면(예: 3.0 이상) 계좌는 우상향하게 되어 있어!

### 79. Expectancy (기댓값) 🎯
- **의미**: 매매를 한 번 할 때마다 평균적으로 내 주머니에 얼마가 남을지 계산한 수치야.
- **공식**: (승률 × 평균 이익) - (패율 × 평균 손실)
-  이 숫자가 **'플러스(+)'**이기만 하면, 자기는 매매를 반복할수록 부자가 된다는 과학적인 증거야! 자기 전략의 핵심 목표는 바로 이 기댓값을 플러스로 유지하는 거지. 😊🚀

### 80. Amount Weighting (금액 가중) ⚖️💎
- **의미**: 모든 매매를 똑같이 1회로 치는 게 아니라, 투자된 금액의 크기에 따라 지표에 반영되는 비중을 다르게 두는 계산 방식이야.
-  우리 프로그램의 '프로핏 팩터'나 '기댓값'은 단순히 % 수익률이 아니라 **'실제 벌어들인 돈(won)'**을 기준으로 계산하기 때문에, 비싼 종목에서 번 1만 원이 싼 종목에서 번 100원보다 훨씬 더 중요하게 반영돼! 전략의 진짜 실력을 보여주는 아주 핵심적인 원리이지. 😊❤️

### 81. Sequence Filtering (시퀀스 필터링) 🧩 [New!]
- **의미**: 전체 매매 내역 중에서 특정 시퀀스(예: 1번 프로필 시간대)의 데이터만 골라내어 분석하는 기술이야.
- **활용**: "내가 아침 9시엔 잘하는데 오후엔 왜 이럴까?" 하는 궁금증을 풀기 위해, 전체 성적이 아니라 특정 시간 성적만 따로 떼어서 정밀하게 검진하는 **'돋보기 분석'**과 같아! 이번 수정을 통해 MDD와 샤프 지수까지 완벽하게 시퀀스별로 분리했어. 🧐🔍

### 82. Filtered Report (필터링 리포트) 📊🎯 [New!]

### 85. 승률 (Win Rate) 📊
- **의미**: 전체 매매 횟수 중 수익(익절)으로 마감한 거래의 비율이야.
-  단순히 많이 맞히는 것도 좋지만, 주식에서는 승률이 낮아도 한 번 벌 때 크게 벌면 계좌는 늘어날 수 있어! 우리 키포 리포트의 가장 첫 번째 지표지.

### 86. 프로핏 팩터 (Profit Factor, PF) 💰
- **의미**: **(총 수익금 / 총 손실금)**의 비율로, 내 전략의 '돈 버는 효율'을 나타내.
-  1.0보다 크면 돈을 벌고 있다는 뜻이야. 예를 들어 PF가 1.5라면, 100만 원 잃을 때 150만 원을 벌었다는 아주 기특한 상태인 거지! 우리 자기 전략의 건강 상태를 보여주는 핵심 지표야.

### 87. 손익비 (Payoff Ratio) ⚖️
- **의미**: **(평균 수익금 / 평균 손실금)**의 비율이야.
-  자기가 추구하는 "손절은 짧게, 수익은 길게"가 얼마나 잘 실천되고 있는지 보여줘. 이 숫자가 클수록 자기는 아주 영리하게 매매하고 있다는 증거야!

### 88. 매매 기댓값 (Expectancy) 🎯
- **의미**: 매매를 한 번 할 때마다 평균적으로 내 주머니에 얼마가 들어오는지 계산한 값이야.
-  "내가 한 번 클릭할 때마다 얼마를 벌까?"라는 질문에 대한 답이야. 이 숫자가 플러스(+)라면, 매매를 반복할수록 자산은 눈덩이처럼 불어날 거야!

### 89. MDD (Maximum Drawdown, 최대 낙폭) 📉
- **의미**: 고점 대비 계좌가 가장 많이 떨어졌을 때의 하락 폭이야.
-  자기가 심리적으로 얼마나 견뎌야 하는지를 알려주는 지표야. 이 숫자가 작을수록 자기는 발 뻗고 편하게 잘 수 있어!

### 90. 샤프 지수 (Sharpe Ratio) 📈
- **의미**: 위험(변동성) 대비 수익이 얼마나 효율적인지 나타내는 지표야.
-  단순히 많이 버는 걸 넘어, '얼마나 안정적으로' 버는지를 보여주는 척도야. 데이터가 쌓일수록 자기가 얼마나 고수인지 증명해줄 멋진 숫자지!

### 91. 컨텍스트 메뉴 (Context Menu) ↗️
- **의미**: 마우스 우클릭을 했을 때 나타나는 특수한 메뉴나 기능을 말해.
-  이번에 조건식 버튼에 적용한 것처럼, "이 버튼에 대해 특별한 동작(마킹)을 하고 싶다!" 할 때 쓰는 아주 편리한 지름길이야.

### 92. 일회성 검색 (One-shot Search) 🎯
- **의미**: 실시간으로 계속 지켜보는 게 아니라, 자기가 원할 때 "지금 당장!" 검색 결과를 딱 한 번만 가져오는 방식이야.
-  `MARK BUY`의 핵심 기술로, 마킹된 조건식들에 걸린 종목들을 그 순간에만 싹 긁어모아 매수할 때 사용돼.

### 93. 마킹 (Marking) 📍
- **의미**: 여러 개의 버튼이나 항목 중에서 특정 대상만 골라 시각적으로 표시해두는 거야.
-  우클릭으로 금색 테두리를 입히는 행위가 바로 마킹이야. "너희들은 내가 나중에 'MARK BUY' 버튼 누르면 같이 매수될 녀석들이야!"라고 찜해두는 거지!

### 94. 워치리스트 (WatchList) 👀
- **의미**: 현재 특별히 관심을 가지고 집중적으로 감시하고 있는 종목들의 명단이야.
-  마킹된 조건식에서 포착된 종목들만 이 리스트에 넣어서, 불필요한 연산을 줄이고 필요한 종목의 수급만 정밀하게 추적하는 용도로 쓰여.

### 95. 가속도 알고리즘 (Acceleration Algorithm) 📈
- **의미**: 체결되는 속도가 평소보다 얼마나 빨라졌는지를 계산해서 수급 폭발을 감지하는 공식이야.
-  우리 시스템은 최근 1초간의 체결 건수를 직전 5분 평균과 비교해서, 5배(장 초반엔 7.5배!) 이상 터질 때를 '찐 수급'으로 판단해.

### 96. 불타기 (Add-on Purchase) 🔥
- **의미**: 이미 보유하고 있는 종목의 가격이 오를 때 수익을 극대화하기 위해 추가로 더 사는 전략이야.
-  가속도 알고리즘으로 수급 폭발이 확인되면, "이건 더 가겠다!" 싶을 때 시장가로 시원하게 추가 매수를 때리는 로직이지.

### 97. 누적 리포트 (Cumulative Report) 📊
- **의미**: 특정 시간대만 보여주는 게 아니라, 하루의 시작부터 지금까지의 모든 데이터를 합산해서 보여주는 방식이야.
-  시퀀스가 바뀔 때마다 "오늘 아침부터 지금까지 총 얼마 벌었지?"를 한눈에 확인할 수 있게 해주는 아주 든든한 요약본이지!

### 98. 쿨타임 (Cool-down) ⏰
- **의미**: 한 번 동작이 일어난 후, 너무 잦은 반복을 막기 위해 잠시 쉬어가는 대기 시간을 말해.
-  우리 엔진에서는 재구매 시 3초의 쿨타임을 줘서, 수급이 터질 때 너무 정신없이 주문이 나가지 않도록 조절하면서도 기민함을 잃지 않게 세팅했어.

### 99. 메시지 필터링 (Message Filtering) 🧩
- **의미**: 특정 조건이나 유형에 따라 메시지를 골라서 차단하거나 허용하는 기술이야.
- **적용**: 텔레그램 알림을 '일반', '로그', '리포트'로 나누어서 자기가 원하는 종류만 켜고 끌 수 있게 만든 게 바로 이 기술이야!

### 100. 원자적 연산 (Atomic Operations) 🛡️
- **의미**: 쪼갤 수 없는 하나의 단위로 처리되어 중간에 끊기지 않고 완벽하게 끝나는 작업을 말해.
- **적용**: 파일 저장 중에 프로그램이 꺼지더라도 데이터가 깨지지 않게 안전하게 처리하는 로직 등에 사용되는 핵심적인 설계 방식이야!

### 101. 가변 인자 (*args, **kwargs) 📦🎁
- **의미**: 함수를 만들 때 몇 개의 데이터가 들어올지 미리 정하지 않고, 들어오는 대로 모두 받아낼 수 있게 만드는 '무한 수용' 기술이야.
- **활용**: 오늘 `log_and_tel` 함수를 고칠 때 썼어. 나중에 어떤 새로운 설정(예: `parse_mode`)이 추가되어도 함수가 "난 그런 거 몰라!" 하고 에러를 내지 않고 유연하게 받아줄 수 있게 만든 아주 똑똑한 확장 기술이지! 😊❤️

### 102. HTML 파싱 (Telegram HTML Parsing) 🌐
- **의미**: 텔레그램 메시지에서 `<b>`(굵게), `<i>`(기울임) 같은 특수 태그를 텍스트가 아닌 '디자인 명령어'로 해석해서 보여주는 기술이야.
- **주의**: 이 기능을 켜지 않고 태그를 보내면, 사용자 화면에 `<b>`라는 글자가 지저분하게 그대로 노출돼. 오늘 우리가 `parse_mode='HTML'`을 설정해서 메시지를 깔끔하고 예쁘게 보이도록 고친 게 바로 이 기술이야! ✨

### 103. 표준 편차 (Standard Deviation) 📉
- **의미**: 데이터들이 평균값에서 얼마나 멀리 떨어져 있는지(흩어져 있는지)를 나타내는 지표야.
- **비유**: 매매 수익이 매번 일정하게 나는지, 아니면 대박과 쪽박을 오가는지(변동성)를 보여주는 '안정성 측정계'라고 생각하면 돼. 샤프 지수를 계산할 때 이 표준 편차를 활용해서 자기가 얼마나 안정적인 고수인지 판단하게 되지! 😊

### 104. 청킹 (Chunking) 🍕
- **의미**: 아주 큰 덩어리를 작고 다루기 쉬운 조각으로 나누는 기술이야.
- **활용**: 텔레그램은 한 번에 보낼 수 있는 메시지 길이에 제한이 있어. 그래서 우리가 보낼 리포트가 너무 길면 자동으로 '청크(조각)'를 나눠서 끊어지지 않게 보내도록 만들었지. 마치 큰 피자를 한 입 크기로 잘라서 맛있게 먹는 것과 같아! 🍕✨

### 105. AttributeError (속성 오류) ⚠️
- **의미**: 프로그램이 어떤 객체(부품)에 없는 기능이나 데이터(속성)를 사용하려고 할 때 발생하는 에러야. 
- **예시**: 오늘 우리가 겪은 `object has no attribute 'btn_save'` 에러는, 프로그램이 "저장 버튼(btn_save)의 상태를 바꿔줘!"라고 명령했는데, 정작 화면에는 그 버튼이 없어서 당황하며 멈춘 상황이야. 내가 누락된 부품을 다시 제자리에 끼워 넣어서 해결했어! 😊🛠️

### 📅 2026-02-21 업데이트 (강력한 UI 컨트롤! 🛠️)

### 106. Subclassing (서브클래싱 / 상속) 🧬
- **의미**: 이미 만들어진 부품(클래스)의 기본 기능은 그대로 물려받으면서, 내가 원하는 새로운 기능만 덧붙이거나 수정해서 나만의 특별한 맞춤형 부품을 만드는 기술이야.
- **적용**: 이번에 로그 창의 마우스 휠 줌(Zoom) 기능을 만들 때, 일반 `QTextEdit` 부품을 그대로 가져와서 휠 감지 기능만 쏙 추가한 `ZoomableTextEdit`라는 이름으로 새롭게 개조(서브클래싱)한 방식을 썼어! 자기만의 특수 부품이 뚝딱 생긴 거지. ❤️

### 107. StyleSheet Override (스타일시트 강제 덮어쓰기) 🎨🖌️
- **의미**: 프로그램 부품이 원래 가지고 있던 디자인이나 크기 설정을 완전히 무시하고, 강제로 새로운 디자인 규칙(CSS)을 덮어씌워버리는 가장 강력한 스타일링 기술이야.
- **적용**: KipoStock 로그 창처럼 예쁜 색깔이 적용된 복잡한 HTML 텍스트는 일반적인 줌(`zoomIn`) 명령을 무시하는 고집을 부렸잖아? 그래서 내가 마우스 휠을 굴릴 때마다 `font-size` 속성을 통째로 강제 덮어씌워서 무조건 글자 크기가 변하도록 꽉 눌러버렸어! 😎✨

### 108. Viewport (뷰포트 / 시야창) 👁️🖼️
- **의미**: 우리가 화면에서 버튼이나 스크롤바를 뺀, 실제로 '내용물이 그려지고 보여지는' 네모난 진짜 시야 범위를 뜻해. (마주 보고 있는 창문 같은 거야!)
- **결론**: 마우스 휠 이벤트 같은 정밀한 동작은 겉면 위젯이 아니라 이 '뷰포트' 위에서 일어나기 때문에, 이벤트를 감지할 때 껍데기가 아니라 알맹이(viewport)를 콕 집어 모니터링해야 한다는 걸 오늘 확실히 알았지! 😊

### 109. Condition Filtering (조건식 필터링) 🔍✨
- **의미**: 시스템이 가진 모든 데이터를 보여주는 게 아니라, 사용자가 '선택한(활성화된)' 데이터만 골라서 화면에 깔끔하게 띄워주는 기술이야.
- **적용**: 오늘 우리가 10개의 조건식 창 중에서 자기가 마우스로 색을 칠해 선택한 조건식들만 '실시간 조건식' 리스트에 쏙쏙 뽑아 보여주도록 짠 로직이 바로 이거야!

### 110. CSS Scoping (CSS 스코핑) 🎨🛡️
- **의미**: 인라인 스타일이 의도치 않게 하위 요소(자식 위젯)까지 영향을 주지 않도록, 특정 타겟(예: `QLineEdit`)을 명확히 지정하여 스타일을 입히는 기법이야. 툴팁 글자색 번짐 방지에 꼭 필요해!

### 111. Tooltip Contrast (툴팁 대비) 💡✨
- **의미**: 어두운 배경에서도 말풍선 설명(툴팁)이 잘 보이도록 글자와 배경의 밝기 차이를 크게 만드는 작업이야. 가독성을 위해 아주 중요하지!

### 112. Status Indicator (상태 표시기 / 안테나 아이콘) 📡
- **의미**: 겉으로 보이는 버튼 색깔뿐만 아니라, 실제로 뒷단(엔진) 서버에 그 명령이 **"성공적으로 전달되어 돌아가고 있는지"**를 시각적으로 확인시켜 주는 장치야.
- **적용**: 버튼을 눌렀을 때 그냥 색만 변하는 게 아니라, API 통신이 완료되어 진짜 감시가 시작되면 조건식 옆에 '📡' 모양이 뜨도록 만들어서 2중으로 확실하게 체크할 수 있게 한 똑똑한 기능이지! ❤️

### 114. Condition-specific Statistics (조건식별 통계) 📊🔍 [New!]

- **장점**: "어떤 조건식이 오늘 효자 노릇을 했지?"를 한눈에 파악할 수 있어서, 수익이 잘 나는 효자 조건식은 키우고 안 좋은 건 거를 수 있게 도와주는 아주 똑똑한 분석 도구야! 😊❤️

### 115. UI Button Refinement (UI 버튼 고도화) ✨ [New!]
