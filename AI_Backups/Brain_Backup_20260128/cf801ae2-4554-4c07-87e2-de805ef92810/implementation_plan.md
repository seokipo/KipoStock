# 알림창 고착 문제 근본 해결 계획 (오버레이 방식)

`QMessageBox`가 특정 상황에서 닫히지 않고 메인 루프를 점유하는 문제를 해결하기 위해, 시스템 창을 띄우지 않고 프로그램 내부 상단에 잠시 나타나는 **오버레이(Overlay)** 방식을 도입합니다.

## 제안된 변경 사항

### 1. [UI] 커스텀 오버레이 라벨 구현
#### [수정] [Kipo_GUI_main.py](file:///d:/Work/Python/AutoBuy/KipoStockNow/Kipo_GUI_main.py)
 - **신규 기술**: `QLabel`을 상속받은 별도의 위젯(또는 메서드 내 동적 생성)을 사용하여, 메인 윈도우의 자식으로 존재하면서 화면 중앙 상단에 플로팅 상태로 메시지를 표시합니다.
 - **작동 방식**:
    - 메시지 전달 시 반투명한 검은 배경에 흰색 글씨로 예쁘게 나타납니다.
    - OS의 포커스를 뺏지 않아 명령어 입력에 방해가 되지 않습니다.
    - `QTimer`에 의해 2초 후 스스로 `hide()` 또는 `deleteLater()` 됩니다.
 - **강점**: 시스템 팝업창이 아니기 때문에 "닫히지 않는 현상" 자체가 발생할 수 없는 가장 안정적인 구조입니다.

### 2. [UI] 적용 및 빌드
 - `on_seq_auto_toggled` 및 다른 알림 구간에 이 오버레이 방식을 전면 적용합니다.

## 검증 계획
 - 시퀀스 버튼 클릭 시 화면 중앙 상단에 "휴장 안내" 메시지가 나타나고 2초 뒤 조용히 사라지는지 확인합니다.
 - 메시지가 떠 있는 동안 하단의 명령어 입력창에 타이핑이 가능한지(입력 방해 여부) 확인합니다.
